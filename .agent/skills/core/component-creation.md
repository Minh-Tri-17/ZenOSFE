# Default Angular Component Creation Skill

This file serves as a system instruction/skill for the AI agent to generate basic, skeletal Angular components using the Angular CLI.

---

## 1. Objective
Ensure that new components are generated quickly, consistently, and remain in a clean, default (skeleton/boilerplate) state. **Do not introduce any business logic or custom UI code** beyond the files generated by the Angular CLI, unless explicitly requested by the user.

---

## 2. Initialization Command
Always execute the following command in the frontend project's root directory:

```bash
ng g c <path> --skip-tests
```

### Key Parameters:
*   **`<path>`**: The path to the new component (e.g., `src/app/shared/components/custom-button` or `shared/components/custom-button`). The CLI automatically creates the directory and the corresponding component files.
*   **`--skip-tests`**: This flag is required to prevent generating the `.spec.ts` unit test file, unless the user has explicitly requested writing unit tests.

---

## 3. Golden Rules & Execution Constraints

> [!IMPORTANT]
> **No Unsolicited Code Additions:**
> Newly generated components must remain exactly as generated by the Angular CLI (containing only the default placeholder template like `<p>name works!</p>`, an empty class, and an empty style file).
> 
> **Under no circumstances should you automatically:**
> 1. Design complex UI layouts (e.g., adding tables, forms, custom buttons, icons).
> 2. Import external modules, services, facades, or other components unless explicitly requested.
> 3. Write business logic, state management, or component methods inside the `.ts` class.

---

## 4. Step-by-Step Workflow
1. **Identify the path**: Determine the target component directory according to the user's requirements.
2. **Run the command**: Execute `ng g c <path> --skip-tests` in the terminal.
3. **Keep it minimal**: Stop modifying the files as soon as the CLI successfully generates them. Do not apply any additional changes to the newly created `.ts`, `.html`, or `.scss` files.
